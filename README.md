# МЕОТИДА — Недостача / План-Факт

Одностраничное HTML-приложение (один файл `index.html`, работает офлайн — просто откройте в браузере).

## Вкладки

1. **План / Факт** — сравнение двух списков кодов, как в исходном приложении
   (`plan_vs_fact.html`): вставили план и факт → «Найти расхождения» → список
   отсутствующих (и лишних) кодов.
2. **Недостача** — пошаговый разбор недостачи в коробе с таможни:
   - шаг 1: коды из письма склада `Tamo data from 185.154.72.113`;
   - шаг 2: «Инфо по марке» из универсальной проверки «Честный знак»
     (товар, МРЦ, ГТД/РД, счёт блоков и пачек);
   - шаг 3: «Состав марки» — план: что должно лежать в коробе;
   - шаги 4–5: «История агрегации» в два этапа (пачка → блок → короб);
   - шаг 6: автоматический расчёт — недостача, каких пачек не хватает в
     разорванных блоках, **яркое выделение чужих блоков и пачек из другого
     короба**;
   - шаг 7: статусы и владельцы отсутствующих блоков (недостача или уже
     продан/списан);
   - шаг 8: готовое письмо о недостаче — кнопка «Написать в Thunderbird»
     (скачивается .eml, Thunderbird открывает заполненное письмо), адресаты
     по умолчанию редактируются, выделенная часть письма — жирная и цветная.

   На каждом шаге — подсказки (куда зайти, что нажать) и стандартное имя
   файла выгрузки с кнопкой «Скопировать имя»
   (например `ChapmanБраун_265_агрегация_1_15072026_2046`).

3. **Как пользоваться** — инструкция с картинками (скриншоты из
   `pipline_2.png`, встроены прямо в файл).

## Сессии

Кнопка «Сохранить текущую сессию» запоминает до 10 последних разборов в
памяти браузера; поиск по сессиям находит код (короб/блок/пачку) в старых
разборах — чтобы увидеть короб, в котором раньше уже были перепутаны блоки
и пачки. Есть выгрузка/загрузка всех сессий в JSON-файл.

## Технически

- один файл, без внешних зависимостей (шрифты системные, картинки — data-URI);
- таблицы вставляются копированием из Excel (Ctrl+A, Ctrl+C → Ctrl+V);
- криптохвосты обрезаются автоматически (по колонке `cis` выгрузок, а без
  выгрузок — эвристикой по структуре кода);
- дизайн — по `DESIGN_SYSTEM.md` (тёмный фон, градиент голубой→розовый→фиолетовый).
